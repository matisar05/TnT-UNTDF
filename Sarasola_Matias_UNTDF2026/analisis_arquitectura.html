Análisis Arquitectónico y Estructural del Proyecto React Native / Expo

A continuación, se detalla la estructura y los archivos principales de este proyecto en React Native (utilizando Expo Router). La explicación es técnica, directa y orientada a entender la justificación arquitectónica detrás de cada decisión en el código.

1. Estructura de Carpetas

La aplicación está contenida íntegramente dentro de la carpeta src/ (Source). Agrupar todo aquí aísla el código fuente de los archivos de configuración de la raíz (como package.json, .gitignore, entre otros).

  • src/app/: Es el corazón de Expo Router. Aquí ocurre el file-based routing (enrutamiento basado en sistema de archivos). Cada archivo .tsx que se coloca aquí se vuelve automáticamente una pantalla navegable. La estructura de carpetas aquí define directamente las URLs internas de la app.
  • src/components/: Los componentes visuales modulares que no son pantallas completas. Se dividen en subcarpetas lógicas (home, layout, categoria, producto) para mantener el orden. Son piezas reutilizables que actúan como bloques de construcción.
  • src/constants/: Almacena variables estáticas e inmutables (como colores, tamaños de espaciado y datos de prueba locales). Separar esto en un módulo constante permite cambiar por completo la apariencia visual del sistema modificando un solo archivo, sin tener que tocar la lógica de los componentes.
  • src/navigation/: Centraliza la lógica direccional, los catálogos o los diccionarios de URL permitidos en la aplicación.

2. Análisis Archivo por Archivo (Línea por Línea clave)

src/navigation/routes.ts

export const ROUTES = {
  HOME: '/',
  CATEGORIA: '/categoria/[slug]',
  // ...
} as const;
  • ¿Qué hace?: Exporta un objeto constante con todas las rutas posibles de la navegación en la aplicación.
  • ¿Por qué está así?: Utiliza sintaxis as const para convertir estas rutas en variables strictas de solo-lectura para TypeScript. Evitamos de este modo usar strings literales ("textos quemados al aire") en nuestros demás componentes. Si en el futuro una ruta cambia de nombre (ej. de /categoria/ a /seccion/), se modifica en un solo lugar y se propaga automáticamente, previniendo links rotos o errores inesperados.

src/constants/tema.ts

export const tema = {
  colors: { primary: '#E67E22', border: '#FAD390', ... },
  spacing: { md: 16, lg: 20, ... },
  text: { base: 13, lg: 16 }
};
  • ¿Qué hace?: Define los conocidos Design Tokens o directrices de sistema de diseño corporativo (todos los colores, radios, márgenes y sombras).
  • ¿Por qué está así?: El desarrollo moderno y limpio prohíbe fuertemente el uso de "Magic Numbers" (números aleatorios fijos en los View como padding: 16). Al utilizar tema.spacing.md, el diseño visual de toda la app se rige sobre una escala estándar y permite a futuro construir temas oscuros/claros sin reemplazar código.

src/app/_layout.tsx

export default function RootLayout() {
  return (
    <Stack screenOptions={{ headerStyle: { backgroundColor: tema.colors.primary } }}>
       <Stack.Screen name="index" options={{ headerShown: false }} />
       <Stack.Screen name="categoria/[slug]" />
    </Stack>
  );
}
  • ¿Qué hace?: Interviene fundamentalmente en todas las pantallas simultáneamente y genera una navegación tipo Stack (historial apilable de pantallas).
  • ¿Por qué está así?: Se aplican screenOptions globalmente para colorear la barra de navegación superior (Header) del color institucional en vez de pintarlo hoja por hoja. Declaramos la subruta <Stack.Screen name="index"> intencionalmente para apagar su header natural con headerShown: false, ya que el Home tiene un UI superior independiente (el HomeHero).

src/app/index.tsx (Pantalla Principal)

export default function HomeScreen() {
  return (
    <View style={styles.container}>
      <ScrollView>
        <HomeHero />
        {/* ... Secciones ... */}
      </ScrollView>
      <TabBar />
    </View>
  );
}
  • ¿Qué hace?: Es la pantalla raíz o '/'. Forma todo el layout estructural principal integrando el módulo de HomeHero arriba y a TabBar inferior.
  • ¿Por qué está así?: Aplica Patrón de Diseño Inteligente: La pantalla se divide en submódulos para no abarrotarse de código. La pantalla "Index" es un Smart Component o ensamblador, en tanto a los componentes que llama son piezas simples. Se emplea un ScrollView envolviendo el contenido central para autorizar su desplazamiento vertical si los elementos del celular exceden su altura en píxeles.

  • ¿Qué hace?: Es una plantilla o pantalla renderizada al vuelo para mostrar los productos de la categoría consultada.

src/components/categoria/TarjetaCategoria.tsx

type TarjetaCategoriaProps = {
  nombre: string;
  slug: string;
  // ...
};

export const TarjetaCategoria: FC<TarjetaCategoriaProps> = ({ nombre, slug }) => {
  const router = useRouter();

  return (
    <Pressable onPress={() => router.push({ pathname: ROUTES.CATEGORIA, params: { slug } })}>
       {/* React Node GUI ... */}
    </Pressable>
  );
};
  • ¿Qué hace?: Una tarjeta interactiva modular para navegar los datos particulares de una categoría.
  • ¿Por qué está así?:
    1. Usamos type de TypeScript para acorazar estructuralmente el diseño de las propiedades e impedir que alguien invoque mal a la tarjeta.
    2. Usa Tipos Estrictos (FC / Functional Component) para el análisis perito del compilador sin fallas en los sub-nodos.
    3. Reemplaza tecnología obsoleta (Touchables) por Pressable, un motor de interacciones más robusto y reaccionario frente al panel táctil. Se navega finalmente usando router.push({ pathname, params }), previniendo construir endpoints concatenados rotos.

src/components/home/HomeHero.tsx

  hero: {
    backgroundColor: tema.colors.primary,
    padding: tema.spacing.lg,
    paddingTop: tema.spacing.xl,
    gap: tema.spacing.sm, // Distribuye los espacios entre subcarpetas
  },
  • ¿Qué hace?: Genera el cuadro informativo naranja ubicado en la azotea de la página.
  • ¿Por qué está así (estilos)?: Emplea utilidades Flexbox modernas de React Native. Se utiliza la propiedad gap: tema.spacing.sm en el nodo raíz; que distribuye matemáticamente distancias exactas de espacios huecos entre todos sus bloques hijos equitativamente. Históricamente tendríamos que haber colocado un marginBottom en cada hijo interior con lógica desordenada, lo cual resulta ineficiente.

  <Text style={[styles.tabLabel, isHome && { color: tema.colors.primary, fontWeight: 'bold' } ]}>
  • ¿Qué hace?: Un dock visible adosado en la porción plana inferior para rotar vistas ("Home", "Listas", "Cámara").
  • ¿Por qué está así?: Examina la presencia persistente del Hook usePathname() para entender secretamente que URL estamos visitando. Luego mediante lógica de cortocircuito (isHome && ...) la aplicamos localmente como prop visual enlazado en corchetes sobre la hoja de estilos: Solo si el archivo temporal confirma ser igual a "HOME", el estado activa las fuentes negritas y el color anaranjado de tema destacando selectivamente la navegación.
