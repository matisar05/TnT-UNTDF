Manual técnico línea por línea de React Native & Expo Router

1. src/navigation/routes.ts

export const ROUTES = {
  HOME: '/',
  LISTAS: '/listas',
  CAMARA: '/camara',
  CATEGORIA: '/categoria/[slug]',
  PRODUCTO: '/producto/[barcode]',
  FAVORITOS: '/favoritos',
} as const;

Desglose Detallado

  • export const ROUTES Declaramos una constante global pública. Es la enciclopedia de hacia dónde se puede viajar.
    ¿Por qué? Se evita hacer variables separadas para empaquetarlo bajo un diccionario unificado.
  • HOME: '/' a FAVORITOS: '/favoritos' Relaciona en formato clave-valor una variable semántica en mayúsculas con la URL del sistema de archivos implícita.
    Justificación Des-acoplamiento técnico: Los botones usarán ROUTES.HOME. Si el cliente requiere que la web cambie a "/inicio", alteras esta única línea y la app entera sobrevivirá intacta en vez de romperse por links directos escritos a mano.
  • as const; Inyecta una "Const Assertion" en TypeScript. Sella atómicamente el objeto.
    TypeScript Sin 'as const', TS interpreta que HOME es un simple string mutable (sobre-escribible). Al agregar la directiva, las rutas quedan petrificadas; su tipo formal ya no es string, sino los literales textológicos en sí mismos.

3. src/app/_layout.tsx

export default function RootLayout() {
  return (
    <Stack
      screenOptions={{
        headerStyle: { backgroundColor: tema.colors.primary },
        headerTintColor: tema.colors.bannerText,
      }}
    >
      <Stack.Screen name="index" options={{ headerShown: false }} />
      <Stack.Screen name="categoria/[slug]" options={{ title: 'Categoría' }} />
      {/* ... demás pantallas */}
    </Stack>
  );
}

Desglose Detallado

  • export default function Expo exige que para ser un Layout (Envoltura base) o página, la función deba exportarse obligatoriamente "por defecto". No admite exports con nombre personalizado.
  • <Stack screenOptions={{ ... }}> El Stack representa la memoria de apilación (pantallas superpuestas) típica de iOS y Android.
    ¿Por qué global? screenOptions dicta las reglas a todas las pantallas interiores simultáneamente. Así coloreamos la barra superior (Header) del Naranja Corporativo una sola vez en lugar de aplicarla en cada archivo independientemente.
  • options={{ headerShown: false }} Dentro de la vista principal "index", desactivamos su barra de Header intencionalmente.
    Decisión Estética Nuestra app Home utiliza un cartel de diseño manual gigante (<HomeHero />) que se vería pisoteado e interrumpido si el sistema le agregase la pequeña barra clásica de arriba.

5. src/app/categoria/[slug].tsx

export default function CategoryScreen() {
  const { slug } = useLocalSearchParams<{ slug: string }>();
  const titulo = slug ? slug.charAt(0).toUpperCase() + slug.slice(1).replace('-', ' ') : 'Categoría';

  return (
    <View>
      <Stack.Screen options={{ title: titulo }} />
      {/* UI ... */}
    </View>
  );
}

Desglose Detallado

  • const { slug } = useLocalSearchParams<{ slug: string }>(); Destrucción algorítmica de Expo Router. Extrae de la URL invisible del teléfono el valor que el usuario presionó.
    Protección Se inserta el tipo en los diamantes para asegurarle al compilador que será puro string previniendo variables Indefinidas.
  • const titulo = ... Saneamiento inteligente. Convierte entradas URL feas ("pastas-frescas") en nombres pulcros capitalizados ("Pastas frescas") reemplazando guiones al vuelo para ahorrar llamadas lentas a Bases de Datos.
  • <Stack.Screen options={{ title: titulo }} />
    Magia en Render Declarado "adentro" de la página, Expo extrae estáticamente la orden y sobreescribe inmediatamente el encabezado blanco de la cabecera del dispositivo reemplazándolo por el nombre modificado dinámicamente.

6. src/app/producto/[barcode].tsx

const { barcode } = useLocalSearchParams<{ barcode: string }>();

Desglose Detallado

  • { barcode: string }
    Riesgo Evitado ¿Por qué un código de barras es tratado como "texto" (string) y no "número entero"? Porque muchos productos (como gaseosas) inician con códigos "0005...". Matemáticamente todos los lenguajes eliminan los ceros a la izquierda en variables number destruyendo el código identificador y estallando bases de datos. Requerirlo como string protege la integridad física de los ceros izquierdos.

7. src/app/listas/index.tsx

    <FlatList
      data={CATEGORIAS}
      keyExtractor={(item) => item.id}
      renderItem={({ item }) => ( ... )}
    />

Desglose Detallado

  • FlatList El componente estructural titánico. No se usan `.map` para listas grandísimas.
    Rendimiento FlatList utiliza el algoritmo "Virtualizado". Si tuvieras 2.000 listados, tu teléfono no tiene 4GB de RAM libre, estallaría. FlatList dibuja solamente 8 en pantalla en ese segundo, y destruye la tarjeta número 1 cuando tus dedos van escrolleando haciendo reciclaje térmico de la memoria logrando una fluidez nativa a 60 FPS inquebrantable.
  • keyExtractor
    Regla de Re-conciliación Exigido por arquitectura virtual de React. Provee a la tarjeta generada de un "Documento de Identidad" único. Si la tarjeta #4 es eliminada por backend, React borrará ese ID único en base sin malgastar CPU en alterar a todas y cada una de las demás.

8. src/components/categoria/TarjetaCategoria.tsx

export const TarjetaCategoria: FC<TarjetaCategoriaProps> = ({ nombre, slug, icono, color }) => {
  const router = useRouter();

  return (
    <Pressable onPress={() => router.push({ pathname: ROUTES.CATEGORIA, params: { slug } })}>
      <View style={[styles.iconContainer, color ? { backgroundColor: color + '20' } : null]}>
        <Ionicons name={icono} size={24} color={color || tema.colors.primary} />
      </View>
      {/* ... */}
    </Pressable>
  );
};

Desglose Detallado

  • FC y Props El componente funcional estricto protege tu ingreso ("Props") obligando a introducir los parámetros mínimos o denegará su compilado.
  • Pressable vs Touchable
    API Moderna Reemplazó a TouchableOpacity en RN 0.63. Te da un log inmersivo con intercepción cruda y el poder de configurar respuestas táctiles sutiles y vibratorias. Se enruta al destino mediante un Objeto de Parámetros {pathname, params} protegiendo a la URL de concatenaciones desprolijas con '+'.
  • color + '20' (Canal Alfa Visual Mod)
    Diseño En código HEX Nativo, el '20' adjuntado al final equivale a "12% de Opacidad / Transparencia" del matiz base. Permite a los programadores teñir levemente el fondo de la etiqueta visual creando nubes apasteladas hermosas aprovechando el mismo color crudo dictado en props y sin exigir la entrega paralela de una versión desaturada del mismo.

11. src/components/layout/TabBar.tsx

  const pathname = usePathname();
  const isListas = pathname.startsWith(ROUTES.LISTAS);

  <Text style={[styles.tabLabel, isHome && styles.activeTabLabel]}>Inicio</Text>

Desglose Detallado

  • .startsWith(ROUTES.LISTAS)
    Lógica de Ruteo No se evalúa matemáticamente con triple igual (`=== '/listas'`). Si el cliente avanza internamente buceando al detalle de lista concreta `/listas/detalles`, el botón interior seguiría reportándose vigente, encendido y con focus sin apagarse, mediante un chequeo de 'comienzo URL'.
  • [styles.tabLabel, isHome && styles.activeTabLabel] Técnica CSS de Array Short Circuit (Cortocircuito). Incorpora estatus reactivo al vuelo: si el ruteo afirma tu posición hogareña, la tabla inserta el texto resaltado por sobre la primera base inerte, tiñéndote condicionalmente a demanda.
